WHY THE PROJECT IS FROZEN HERE

Alpha.25 is a coherent milestone: the product contract, boot paths, state authority, configuration revisions, firstboot, runtime rendering, huge-page preparation, systemd miner ownership, operator command and health evidence exist together.

Freezing the milestone protects that record. It prevents later work from silently rewriting what was actually proven at Alpha.25. Future development can move forward, but it should do so from a named, inspectable baseline.

NO MARKETING ESCAPE HATCH

  • RIGOS does not claim superiority over other products without comparative physical evidence.
  • It does not call acceptance targets “benchmarks” merely because numbers exist.
  • It does not treat source archives as bootable release images.
  • It does not erase upstream XMRig donation behavior.
  • It does not convert “worked once” into “safe unattended forever.”

THE NEXT HONEST GATES

The repository already names the missing work. A future stable claim must be grounded in those completed campaigns and in published evidence that can be inspected independently.
