SYSTEMD ORDERING

  -> rigos-state-ready.service
  -> rigos-ssh-hostkeys.service
  -> rigos-profile-apply.service
  -> rigos-runtime-render.service
  -> rigos-hugepages.service
  -> rigos-miner.service

The miner is not the center of the graph. It is the last privileged consumer of state, configuration, runtime and kernel preparation that must already be true.

CONFIGURATION REVISION MODEL

operator input
    -> validate complete candidate
    -> create immutable revision
    -> commit revision
    -> atomically switch current pointer
    -> apply machine profile
    -> render private runtime config
    -> publish public redacted status
    -> permit service activation

Direct edits to live miner configuration are not the durable authority. The current committed revision is. This makes rollback and evidence collection possible without treating a half-written runtime file as truth.

SECURITY BOUNDARIES

  • Persistent state is verified before mounting and is separate from the immutable root.
  • Raw private runtime configuration, wallet identity, passwords, API token contents and SSH private keys are not public operator output.
  • Recovery defaults to observation. Mutation requires an explicit command.
  • Release artifacts require SHA256 verification. GitHub source archives are not bootable images.
  • The model is practical Alpha security engineering, not a formal certification.
